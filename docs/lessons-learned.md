# Lessons Learned — ai-platform-engineering-portfolio

- Keeping a single landing repo separate from the individual case studies made it much easier to present a coherent narrative in an interview without re-explaining the whole portfolio structure each time.
- Writing the 30/60/90 roadmap alongside the repos (rather than after) helped clarify which case study should go the deepest (the tool registry lab).
- Treating 'what is intentionally excluded' as a first-class section made the sanitization approach explicit and defensible rather than something to explain reactively.
